# QC Lead Analyser — Design Specification

**Date:** 23Mar2026
**Status:** Approved
**App Title:** QC Lead Analyser

---

## 1. Overview

Single-page HTML application with Node.js ESM server for discovering, assessing, and managing aviation business leads across seven geographic regions. The app identifies potential consulting opportunities for QC Aviation Group Pty Ltd by scanning aviation news feeds, regulator publications, and tender sites, then grades each lead against QC's ten service lines.

### QC Service Lines

1. Airline Startup — Domestic and International
2. Aircraft Entry Into Service
3. Sourcing Aircraft — Business Jet and Airline Types
4. Airline Flight Training Consulting
5. Flight Operations Management Consulting
6. Flight Training Simulators — Assessment, Validation and Installation
7. Airline Route Analysis
8. Flight Operations Technology Consulting
9. Auditing — Internal, Regulatory Compliance, IOSA Compliance
10. Flight Operations Manual Suite Construction, Assessment and Amendment

### Geographic Regions

- Australia
- Oceania
- South East Asia
- East Asia
- Central Asia
- Middle East
- Europe

## 2. Architecture

### Files

| File | Purpose |
|------|---------|
| `QC_Lead_Analyser.html` | Frontend — single HTML file with inline CSS/JS |
| `QC_Lead_Analyser_Server.mjs` | Node.js ESM server on port 3849 |
| `QC_Lead_Analyser_data.json` | Discovered leads (Discover tab) |
| `QC_Lead_Analyser_keep.json` | Kept leads with contacts (Keep tab) |
| `QC_Lead_Analyser_archive.json` | Archived leads with notes (Archive tab) |

All files reside in `Aviation_Tools/`.

### Lead ID Generation

Each lead receives a deterministic ID generated by hashing `source_url + title` using the same `hashString()` function used in the AD/SB server. Format: `lead_${hash}`. This ensures:
- Same article from the same source always produces the same ID (deduplication)
- IDs are stable across server restarts
- No collision risk for distinct articles

## 6. Data Flow — FIND Phase

### Trigger

Update button on Discover tab. Only scans regions that have their pills selected.

### Data Sources

**Aviation News RSS Feeds:**
- AeroTime (`https://www.aerotime.aero/rss`)
- Aviation Week (`https://aviationweek.com/rss/air-transport`)
- FlightGlobal (`https://www.flightglobal.com/rss`)
- ch-aviation (`https://www.ch-aviation.com/portal/news/rss`)
- Simple Flying (`https://simpleflying.com/feed/`)
- Aviation Business ME (`https://www.aviationbusinessme.com/feed/`)
- Australian Aviation (`https://australianaviation.com.au/feed/`)
- Airways Magazine (`https://www.airwaysmag.com/feed/`)

**Regulator RSS/API Feeds (where available):**
- CASA media releases: `https://www.casa.gov.au/news-and-media/rss.xml` (if available; fall back to scraping `/news-and-media` listing page)
- EASA news: `https://www.easa.europa.eu/en/newsroom/rss` (if available; fall back to scraping newsroom listing)

**Tender Site:**
- TendersOnTime: `https://www.tendersontime.com/searchrfp/global-aviation-consultancy-rfp-733/`
- Parsing strategy: extract tender listing items from HTML (look for repeated item containers with title, deadline, description). Each tender becomes a lead with entity name derived from the tender title, source = "TendersOnTime", and description from the tender summary. If the page structure changes and parsing fails, log error and skip gracefully.

### Processing Pipeline

1. Fetch all feeds concurrently with 15-second timeout per feed
2. Parse RSS/Atom XML using lightweight regex parser (same pattern as New Airlines server)
3. Parse TendersOnTime HTML for tender listings
4. Deduplicate against existing leads in all three JSON files (match by deterministic ID — same `source_url + title` hash produces the same ID; also reject if normalised title substring of first 40 chars matches any existing lead title)
5. Classify each item by region using keyword map (expanded from New Airlines region map to include Central Asia and East Asia)
6. Filter for relevance using keyword set covering all 10 QC service lines
7. Items that match no region or no service line keywords are discarded
8. Broadcast progress via SSE
9. Failed feeds are logged to console and skipped silently — a summary of failed sources is included in the SSE completion event (`failedSources: [...]`)
10. Scan results are cached for 10 minutes — repeated clicks of Update within the cache window return cached results without re-fetching

| Service Line | Keywords |
|-------------|----------|
| Airline Startup | new airline, startup airline, airline launch, AOC granted, air operator certificate, inaugural flight, commences operations |

## 7. Data Flow — ASSESS Phase (Hybrid)

### Step 1: Rule-Based Scoring (Automatic)

For each discovered news item:
1. Match item text against all service line keyword sets
2. Count matching service lines and keyword hit density
3. Assign grade:
   - **GREEN** (Good Fit): 3+ service lines matched, or strong primary match (multiple keywords in one critical service line like Airline Startup or Simulators)
   - **YELLOW** (Possible Fit): 1-2 service lines matched
   - **AMBER** (Poor Fit): Weak or tangential match only
4. Generate reasoning text: "Matches [service lines]. [Brief explanation of why this creates consulting opportunity]."

### Step 2: AI Deep Assess (Optional, via right-click)

- Triggered by "AI Deep Assess" in context menu
- Requires `ANTHROPIC_API_KEY` environment variable
- If API key not set, menu item appears greyed out with tooltip "Set ANTHROPIC_API_KEY to enable"
- Server sends POST to Claude API with:
  - News item title, description, source
  - QC service line descriptions
  - Prompt requesting: grade (GREEN/YELLOW/AMBER), reasoning, matched service lines
- Response overwrites rule-based grade, reasoning, and service tags on the tile
- Assessment source indicator changes from "Rule-based" to "AI-assessed"

**AI Prompt Template:**
```
System: You are an aviation business development analyst for QC Aviation Group Pty Ltd,
a flight operations consultancy. Assess the following news item as a potential business lead.

QC Aviation offers these services:
1. Airline Startup (domestic and international)
2. Aircraft Entry Into Service
3. Sourcing Aircraft (business jet and airline types)
4. Airline Flight Training Consulting
5. Flight Operations Management Consulting
6. Flight Training Simulators (assessment, validation, installation)
7. Airline Route Analysis
8. Flight Operations Technology Consulting
9. Auditing (internal, regulatory compliance, IOSA compliance)
10. Flight Operations Manual Suite (construction, assessment, amendment)

Respond with JSON only:
{
  "grade": "GREEN|YELLOW|AMBER",
  "gradeLabel": "Good Fit|Possible Fit|Poor Fit",
  "reasoning": "2-3 sentence explanation",
  "matchedServices": ["service name", ...]
}

User: Title: {title}
Source: {source}
Date: {date}
Description: {description}
```

## 8. Keep Tab — Contact Enrichment

### Automatic (on move to Keep)

- Server attempts to find entity website URL as best-effort: constructs search query from entity name, checks if the source article URL contains a domain that looks like the entity's own site
- This is heuristic and unreliable — the website field is left blank if no confident match is found
- Manual entry is the primary mechanism for contact enrichment

### Manual Enrichment (editable fields on Keep tile)

Each Keep tile extends the base card with editable fields:
- **Website** (pre-populated if auto-found, editable)
- **Contact Name** (text input)
- **Contact Email** (text input)
- **Contact Phone** (text input)
- **LinkedIn** (text input)
- **Notes** (text area)

All fields save to `QC_Lead_Analyser_keep.json` on blur (debounced 500ms).

### Keep Tab Update Button

- Triggers `POST /api/keep/update` endpoint
- Server re-fetches the same RSS feeds used by the Discover scan
- Filters articles by matching kept entity names as keywords against article title and description
- New matching articles are appended to the `recentUpdates` array within each keep lead in `QC_Lead_Analyser_keep.json`
- Updates shown as a chronological list on each tile: date, headline, source, link
- Progress reported via SSE (same stream as Discover scan)
- "Last updated" timestamp displayed in Keep tab header

### Keep Tab Layout

Leads grouped under collapsible region headers:
- Region name with lead count
- Click to expand/collapse
- Leads sorted by grade (GREEN first, then YELLOW, then AMBER) within each region

## 12. API Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/` | Serve `QC_Lead_Analyser.html` |
| GET | `/events` | SSE stream for scan progress |
| GET | `/api/discover` | Return discovered leads (with optional region filter) |
| GET | `/api/keep` | Return kept leads |
| GET | `/api/archive` | Return archived leads |
| POST | `/api/scan` | Trigger scan for selected regions (body: `{ regions: [...] }`) |
| POST | `/api/keep` | Move lead to Keep (body: `{ leadId }`) |
| POST | `/api/archive` | Move lead to Archive (body: `{ leadId, notes }`) |
| PUT | `/api/keep/:id` | Update Keep lead fields (contacts, notes) |
| POST | `/api/restore` | Restore archived lead to Keep (body: `{ leadId }`) |
| DELETE | `/api/archive/:id` | Permanently delete archived lead |
| POST | `/api/ai-assess` | AI Deep Assess a lead (body: `{ leadId }`) |
| POST | `/api/keep/update` | Re-scan news for kept entities |

## 13. Security & Assumptions

- **Localhost only** — no authentication, no CORS restrictions. The server binds to `127.0.0.1:3849` and is intended for single-user local use only.
- **No sensitive data in feeds** — all data sources are publicly available RSS feeds and web pages.
- **API key security** — `ANTHROPIC_API_KEY` is read from environment variable only, never stored in JSON files or logged.
